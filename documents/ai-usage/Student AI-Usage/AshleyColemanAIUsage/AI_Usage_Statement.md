# AI Usage Statement

## Overview
This project was developed with the assistance of AI tools to support learning, debugging, and code documentation. All core design decisions, implementation logic, and integration were completed by the developer.

---

## How AI Was Used

### 1. Code Understanding & Learning
AI was used to:
- Explain programming concepts (e.g., pathfinding, event systems, state machines)
- Break down existing code into simpler terms
- Help understand how different components interact

---

### 2. Debugging Support
AI assisted in:
- Identifying logic errors
- Explaining runtime issues
- Suggesting fixes for bugs

All fixes were reviewed, tested, and integrated manually.

---

### 3. Code Documentation
AI was used to:
- Add comments and JavaDoc-style documentation
- Improve readability and structure of scripts

---

### 4. Minor Code Suggestions
AI provided:
- Small code snippets or improvements
- Suggestions for structuring systems

However:
- All code was integrated and adapted by the developer

---

## What Was Done Independently

- Designed the game concept and mechanics
- Built the overall system architecture
- Implemented and connected all scripts
- Tested and debugged gameplay behavior

---

## Reflection

AI was used as a learning tool, not a replacement for development.  
This project reflects my understanding and implementation of the system.

---

## Conclusion

All final code and decisions were understood and implemented by the developer.
